# 萨卡兹语大模型微调计划 (Sarkaz LLM Fine-Tuning Plan)

本文档旨在规划如何利用大型语言模型（如 Qwen 小模型）的强大语境能力，解决《明日方舟：终末地》中“萨卡兹语”到中文的解码与翻译问题。

## 1. 背景与核心挑战

萨卡兹语采用极其简单但信息损耗极大的映射规则：`ord(char) % 56`。
这意味着数以千计的常用中文字符被压缩到了 26 个英文字母中，存在海量的“哈希碰撞”。传统的基于字典或局部概率的模型无法有效应对这种严重的歧义。

**核心解决方案**：引入具备“世界知识”和长程上下文感知能力的 LLM，通过上下文语境进行极端的语义消歧。

## 2. 阶段一：语料整理与平行数据集构建

高质量的语料是微调成功的基础。我们需要构建规模大且富有“终末地”语境的平行语料库。

*   **原始数据源**：
    *   项目现有的 `wordlist.txt`（常用词）、`single_char.txt`（常用字）。
    *   现有的 `wiki2019zh.json`，提供开放域的语言知识。
    *   提取的《明日方舟》及《明日方舟：终末地》剧情文案、道具描述（高优先级）。
*   **平行语料合成**：
    *   利用核心算法生成 `(中文, 萨卡兹语)` 对。
    *   引入噪声：在生成的部分萨卡兹语中人为增加轻微的拼写错误或缺失，增强模型在 OCR 提取场景下的鲁棒性。
*   **SFT (指令微调) 格式构建**：
    构建类似如下的 Prompt-Response 格式：
    ```json
    {
      "instruction": "请结合《明日方舟：终末地》的世界观，将以下萨卡兹密文翻译为中文原意。",
      "input": "ytqqrvqbsbtjyrernx",
      "output": "你好，管理员，是来提交活动菜品的吗？"
    }
    ```

## 3. 阶段二：Tokenizer 的重构与优化

这是最关键的一步，决定了模型是“真正理解词义”还是“盲目猜测字母组合”。
现有的开源模型 Tokenizer 在遇到 `ytqq` 这样的输入时，倾向于按不成逻辑的英文子词进行切片，这会彻底破坏萨卡兹语与原始中文字符形态的对应关系。

*   **中英混合联合训练**：
    *   **不能仅用萨卡兹语训练 Tokenizer**。必须将**原始中文文本**与**对应的萨卡兹语文本**混合，作为语料喂给 BPE (Byte-Pair Encoding) 或 WordPiece 训练器。
    *   利用原始中文语料中丰富的词汇边界统计规律，强制 Tokenizer 学习到“管理员”及其映射 `sbt` 应该作为一个完整的 Token 存在。
*   **领域字典注入**：
    *   读取 `LLMTools/MCPServer/endfield_words.txt` 及其对应的萨卡兹编码，将其作为强制保留的特殊 Token (User-defined symbols) 注入到词表中，确保关键名词不会被切碎。

## 4. 阶段三：模型微调策略 (Fine-Tuning)

*   **基座模型选择**：选择 Qwen 等对中文和多语言支持极好，且具备小参数量版本（如 0.5B, 1.8B 等）的模型，以便在消费级硬件上进行研究。
*   **微调方式**：
    *   初期推荐使用 **LoRA / QLoRA** 进行参数高效微调 (PEFT)，快速验证可行性。
    *   如果 Tokenizer 变动极大（新增了大量词汇），则需要先解冻并训练 Embedding 层，然后再进行 LoRA 微调。
*   **思维链 (Chain of Thought, CoT) 尝试**：
    为了激发极限的语境推理，可以构建部分引导性数据，让模型在给出最终结果前，先输出思考过程：
    *   *模型输出示例*：`[思考] 输入包含片段 sbt，在终末地语境中可能是“管理员”。结合开头 ytqq (你好)，这是日常对话问候。 [翻译] 你好，管理员...`

## 5. 阶段四：评估、验证与部署

*   **评估指标 (Metrics)**：
    *   由于是一对一映射，传统的 BLEU 或 ROUGE 可能不够直观。可以计算**字级别准确率 (Character Error Rate, CER)**。
*   **模型集成**：
    *   微调完成后，将新模型通过 vLLM 或 llama.cpp 在本地部署。
    *   更新现有的 MCP Server（`Sarkaz_tools.py`），加入调用本地微调模型的工具接口，真正实现“AI 与定制化工具协作翻译萨卡兹语”。
